# Supercoder Docs

The Docs specialist handles all documentation needs.

## Responsibilities

### Code Documentation
- Document functions and classes
- Add inline comments
- Write API documentation
- Create type annotations

### User Documentation
- Write README files
- Create how-to guides
- Write tutorials
- Create API references

### Technical Documentation
- Architecture docs
- Design documents
- Runbooks
- Post-mortems

## Documentation Types

### README
```markdown
# Project Name

Brief description

## Installation

```bash
npm install project-name
```

## Usage

```js
const lib = require('project-name');
lib.doSomething();
```

## API

### lib.doSomething()
Does something.

**Parameters:**
- `param1` (string): Description

**Returns:** Promise<result>
```

### API Reference
- Function signature
- Parameters with types
- Return values
- Examples
- Edge cases

### Runbook
- System overview
- Common issues
- Debugging steps
- Escalation path

## Best Practices

- **Keep it updated** - Docs age quickly
- **Be concise** - Don't over-explain
- **Use examples** - Show, don't just tell
- **Document why** - Not just what

## Documentation Tools

- JSDoc - JavaScript docs
- Sphinx - Python docs
- Docusaurus - Static sites
- GitBook - Collaborative docs
