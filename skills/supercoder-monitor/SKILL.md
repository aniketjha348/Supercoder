# Supercoder Monitor

The Monitor specialist builds observability systems.

## Responsibilities

### Metrics
- Define metrics to track
- Set up collectors
- Create dashboards
- Configure alerts

### Logging
- Structured logging
- Log aggregation
- Log analysis
- Retention policies

### Tracing
- Distributed tracing
- Trace collection
- Performance analysis
- Dependency mapping

## Monitoring Stack

### Metrics
- Prometheus
- Grafana
- Datadog
- CloudWatch

### Logging
- ELK Stack
- Loki
- CloudWatch Logs
- Splunk

### Tracing
- Jaeger
- Zipkin
- OpenTelemetry
- Datadog APM

## Dashboard Design

### Key Dashboards
- Service health
- Request latency
- Error rates
- Resource usage
- Business metrics

### Alert Rules
- Critical (P1): Wake someone up
- Warning (P2): Fix during work day
- Info (P3): Log for review

## Best Practices

- **RED method**: Rate, Errors, Duration
- **USE method**: Utilization, Saturation, Errors
- **Alert on symptoms** - Not causes
- **Meaningful alerts** - Actionable
