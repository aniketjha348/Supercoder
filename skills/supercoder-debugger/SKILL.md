# Supercoder Debugger

The Debugger investigates and fixes issues systematically.

## Responsibilities

### Issue Investigation
- Reproduce bugs
- Identify symptoms
- Trace through code
- Find root cause

### Bug Fixing
- Implement fixes
- Test solutions
- Prevent regressions
- Document issues

### Root Cause Analysis
- Use systematic approach
- Trace upstream causes
- Identify patterns
- Prevent recurrence

## Debugging Process

### 1. Reproduce
1. Gather bug report
2. Identify reproduction steps
3. Create minimal reproduction
4. Verify bug exists

### 2. Diagnose
1. Collect relevant logs
2. Add debug output
3. Trace execution
4. Identify root cause

### 3. Fix
1. Implement solution
2. Test fix works
3. Verify no regressions
4. Document the fix

### 4. Prevent
1. Add test coverage
2. Check for similar issues
3. Document lessons learned

## Debugging Techniques

### Binary Search
- Isolate problem area
- Test halfway point
- Narrow down location

### Rubber Ducking
- Explain problem aloud
- Often reveals solution

### Time Boxing
- Set investigation limit
- Take breaks
- Try different approach

### Instrument
- Add logging
- Use debugger
- Profile performance

## Common Bug Patterns

- Null/undefined handling
- Race conditions
- Off-by-one errors
- Type coercion issues
- Async timing issues

## Best Practices

- **Reproduce first** - Can't fix what you can't see
- **One change at a time** - Isolate variables
- **Document findings** - Help future debugging
- **Add tests** - Prevent regression
