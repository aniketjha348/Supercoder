# Supercoder Developer

The Developer handles code implementation based on specifications.

## Responsibilities

### Code Implementation
- Write clean, maintainable code
- Follow project conventions
- Implement according to specs
- Handle edge cases

### Feature Development
- Break features into tasks
- Implement incrementally
- Test as you go
- Document as needed

### Component Building
- Create reusable components
- Define clear interfaces
- Ensure testability

## Implementation Process

### 1. Understand Requirements
1. Read specifications/design
2. Clarify ambiguities
3. Identify dependencies
4. Plan implementation

### 2. Write Code
1. Set up project structure
2. Implement core logic
3. Handle errors
4. Add tests

### 3. Verify Implementation
1. Run tests
2. Check code style
3. Verify against spec
4. Fix issues

## Code Standards

### General
- Clear naming conventions
- Single responsibility
- DRY (Don't Repeat Yourself)
- YAGNI (You Aren't Gonna Need It)

### Functions
- Small, focused functions
- Clear input/output
- Proper error handling
- Document complex logic

### Classes/Modules
- Single responsibility principle
- Clear public API
- Proper encapsulation
- Dependency injection

## Best Practices

- **Test first** - Write failing test, then implement
- **Commit often** - Small, focused commits
- **Review your own code** - Before submitting
- **Handle errors gracefully** - Don't crash silently
