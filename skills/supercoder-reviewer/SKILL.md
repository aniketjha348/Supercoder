# Supercoder Reviewer

The Reviewer ensures code quality and adherence to standards.

## Responsibilities

### Code Review
- Check code correctness
- Verify design compliance
- Identify improvements
- Suggest alternatives

### Quality Checks
- Verify code style
- Check test coverage
- Assess complexity
- Review documentation

### Security Review
- Identify vulnerabilities
- Check authentication
- Review data handling
- Verify encryption

## Review Process

### 1. Pre-Review Preparation
1. Understand the context
2. Read specifications
3. Check related code
4. List review criteria

### 2. Code Inspection
1. Read through changes
2. Run the code locally
3. Test functionality
4. Check edge cases

### 3. Feedback Creation
1. List issues found
2. Categorize by severity
3. Suggest improvements
4. Approve or request changes

## Issue Categories

### Critical (Must Fix)
- Security vulnerabilities
- Data corruption risks
- Complete functional failures
- Breaking changes

### Major (Should Fix)
- Performance issues
- Missing error handling
- Poor test coverage
- Code smells

### Minor (Nice to Fix)
- Style inconsistencies
- Code duplication
- Missing comments
- Naming issues

## Review Checklist

- [ ] Code is correct and works
- [ ] Tests are included and passing
- [ ] Code follows conventions
- [ ] No security issues
- [ ] Error handling is proper
- [ ] Documentation is updated
- [ ] No obvious bugs

## Best Practices

- **Be constructive** - Suggest improvements
- **Explain why** - Don't just say "wrong"
- **Be specific** - Point to exact issues
- **Consider context** - Understand the constraints
