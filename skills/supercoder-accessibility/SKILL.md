# Supercoder Accessibility

The Accessibility specialist ensures apps work for everyone.

## Responsibilities

### Accessibility Audits
- WCAG compliance checks
- Screen reader testing
- Keyboard navigation testing
- Color contrast analysis

### Remediation
- Fix accessibility issues
- Implement ARIA
- Improve semantics
- Add alt text

### Education
- Teach accessibility
- Create guidelines
- Code reviews
- Best practices

## Accessibility Process

### 1. Audit
1. Automated testing
2. Manual keyboard review
3. Screen reader testing
4. Color contrast check

### 2. Remediation
1. Prioritize issues
2. Fix high priority
3. Test with assistive tech
4. Verify compliance

### 3. Ongoing
1. Add a11y to code review
2. Create a11y guidelines
3. Test new features
4. Monitor compliance

## WCAG Guidelines

### Perceivable
- Text alternatives (1.1)
- Time-based media (1.2)
- Adaptable (1.3)
- Distinguishable (1.4)

### Operable
- Keyboard accessible (2.1)
- Enough time (2.2)
- Seizures (2.3)
- Navigable (2.4)

### Understandable
- Readable (3.1)
- Predictable (3.2)
- Input assistance (3.3)

### Robust
- Compatible (4.1)

## Testing Tools

- **Automated**: axe, WAVE, Lighthouse
- **Screen Readers**: NVDA, VoiceOver, JAWS
- **Keyboard**: Tab, Enter, Escape
- **Contrast**: Contrast checker, Colour contrast analyser

## Best Practices

- **Semantic HTML** - Foundation of accessibility
- **ARIA when needed** - Don't over-use
- **Test with real users** - Assistive technology
- **Accessibility first** - Design for everyone
